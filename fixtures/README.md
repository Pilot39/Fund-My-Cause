# Test Fixtures and Seed Data

This directory contains test fixtures and seed data for local development and testing.

## Overview

Fixtures provide realistic, deterministic test data for:
- **E2E tests** - Playwright tests use fixtures for consistent test scenarios
- **Local development** - Developers can work with realistic data without deploying contracts
- **Component testing** - UI components can be tested with various data states
- **Integration testing** - Backend services can be tested with predictable data

## Files

### `test-fixtures.json`
Generated TypeScript fixtures containing:
- **Campaigns** - 10 campaigns covering all lifecycle states
- **Contributors** - Sample contributor profiles
- **Contributions** - Individual contribution records
- **Metadata** - Summary statistics

**Generate with:**
```bash
npx ts-node scripts/generate-fixtures.ts
```

### `seed-data.json`
Generated by seed scripts containing actual deployed contract IDs on testnet:
- Campaign contract addresses
- Registry contract address
- Network configuration
- Deployment timestamps

**Generate with:**
```bash
# Unix/Linux/Mac
./scripts/seed-testnet.sh --creator $CREATOR --token $TOKEN

# Windows
.\scripts\seed-testnet.ps1 -Creator $env:CREATOR -Token $env:TOKEN
```

## Using Fixtures in Tests

### E2E Tests (Playwright)

```typescript
import fixtures from '../fixtures/test-fixtures.json';

test('displays active campaigns', async ({ page }) => {
  // Mock API with fixture data
  await page.route('**/api/campaigns', route => {
    route.fulfill({ json: fixtures.campaigns });
  });
  
  await page.goto('/campaigns');
  
  const activeCampaigns = fixtures.campaigns.filter(c => c.status === 'Active');
  expect(await page.locator('[data-testid="campaign-card"]').count())
    .toBe(activeCampaigns.length);
});
```

### Component Tests (Jest/Vitest)

```typescript
import { render, screen } from '@testing-library/react';
import fixtures from '../../fixtures/test-fixtures.json';
import { CampaignCard } from './CampaignCard';

test('renders campaign card', () => {
  const campaign = fixtures.campaigns[0];
  render(<CampaignCard campaign={campaign} />);
  
  expect(screen.getByText(campaign.title)).toBeInTheDocument();
  expect(screen.getByText(/5%/)).toBeInTheDocument(); // Progress
});
```

### API Integration Tests

```typescript
import fixtures from '../fixtures/test-fixtures.json';

describe('Campaign API', () => {
  it('filters campaigns by status', () => {
    const activeCampaigns = fixtures.campaigns.filter(
      c => c.status === 'Active'
    );
    expect(activeCampaigns.length).toBeGreaterThan(0);
  });
});
```

## Seeding Testnet

The seed scripts deploy actual campaigns to Stellar testnet for integration testing and development.

### Prerequisites

1. **Stellar CLI** installed and configured
   ```bash
   stellar --version
   ```

2. **Testnet account** with XLM balance
   ```bash
   stellar keys generate --network testnet --name testnet-deployer
   ```

3. **Token contract** deployed (or use native XLM)

### Basic Usage

**Unix/Linux/Mac:**
```bash
# Set environment variables
export CREATOR="GBBD47UZQ2QDAAK63XUIFH5FXVLNFSMQC4MLR4LHPWKFG7FMKGV2FI2QI"
export TOKEN="CAQCBGVLWCSMNRMQ4ZBVWWV5CSTW2UNKHJ42OU7CW7ADGNBORAEOSON"

# Run seed script
./scripts/seed-testnet.sh

# Or with inline arguments
./scripts/seed-testnet.sh --creator $CREATOR --token $TOKEN --num-campaigns 10 --verbose
```

**Windows:**
```powershell
# Set environment variables
$env:CREATOR = "GBBD47UZQ2QDAAK63XUIFH5FXVLNFSMQC4MLR4LHPWKFG7FMKGV2FI2QI"
$env:TOKEN = "CAQCBGVLWCSMNRMQ4ZBVWWV5CSTW2UNKHJ42OU7CW7ADGNBORAEOSON"

# Run seed script
.\scripts\seed-testnet.ps1

# Or with inline arguments
.\scripts\seed-testnet.ps1 -Creator $env:CREATOR -Token $env:TOKEN -NumCampaigns 10 -Verbose
```

### Advanced Options

```bash
# Use existing registry
./scripts/seed-testnet.sh --registry-id CXYZ... --creator $CREATOR --token $TOKEN

# Seed mainnet (use with caution!)
./scripts/seed-testnet.sh --network mainnet --creator $CREATOR --token $TOKEN

# Create many campaigns for load testing
./scripts/seed-testnet.sh --num-campaigns 50 --creator $CREATOR --token $TOKEN
```

### Output

After seeding, you'll have:

1. **`.env.local`** - Updated with contract IDs
   ```env
   NEXT_PUBLIC_REGISTRY_ID=CABC...
   NEXT_PUBLIC_NETWORK=testnet
   NEXT_PUBLIC_TOKEN_ID=CDEF...
   NEXT_PUBLIC_CAMPAIGN_0_ID=CXYZ...
   NEXT_PUBLIC_CAMPAIGN_1_ID=CABC...
   ```

2. **`fixtures/seed-data.json`** - Deployment metadata
   ```json
   {
     "generated": "2026-06-28T10:30:00Z",
     "network": "testnet",
     "registry_id": "CABC...",
     "campaigns": [...]
   }
   ```

3. **Deployed contracts** on testnet ready for testing

## Simulating Contributions

The seed scripts create campaigns but cannot automatically add contributions (requires multiple funded accounts). To add contributions manually:

```bash
# Contribute to a campaign
stellar contract invoke \
  --id CAMPAIGN_ID \
  --network testnet \
  --source CONTRIBUTOR_ADDRESS \
  -- contribute \
  --contributor CONTRIBUTOR_ADDRESS \
  --amount 10000000 \
  --token TOKEN_ID \
  --message "Great cause!"
```

### Multi-Account Contribution Script

For realistic testing with multiple contributors, fund several testnet accounts and use a contribution script:

```bash
# Create multiple testnet accounts
for i in {1..5}; do
  stellar keys generate --network testnet --name "testnet-contributor-$i"
done

# Fund accounts from friendbot
# Then contribute from each account
```

## Cleaning Up

### Remove Local Fixtures
```bash
rm fixtures/test-fixtures.json fixtures/seed-data.json
```

### Remove Environment Config
```bash
rm apps/interface/.env.local
# Restore from backup if needed
mv apps/interface/.env.local.backup apps/interface/.env.local
```

### Archive Old Seed Data
```bash
mkdir -p fixtures/archive
mv fixtures/seed-data.json fixtures/archive/seed-data-$(date +%Y%m%d).json
```

## Fixture Schema

### Campaign Fixture
```typescript
interface CampaignFixture {
  id: string;                    // Contract ID
  title: string;                 // Campaign title
  description: string;           // Full description
  creator: string;               // Creator address
  goal: number;                  // Goal in stroops
  deadline: number;              // Unix timestamp
  minContribution: number;       // Min contribution in stroops
  maxContribution: number;       // Max contribution (0 = unlimited)
  status: CampaignStatus;        // Current status
  category: CampaignCategory;    // Category
  currentTotal: number;          // Current raised amount
  contributorCount: number;      // Number of unique contributors
  progress: number;              // Progress percentage (0-100+)
  state: string;                 // Human-readable state
  socialLinks?: string[];        // Social media links
  imageUrl?: string;             // Campaign image
  tags?: string[];               // Search tags
}
```

## Best Practices

1. **Regenerate fixtures** before running test suites to ensure fresh data
2. **Version fixtures** in git for consistent CI/CD testing
3. **Use seed scripts** for integration testing with real contracts
4. **Keep fixtures realistic** - match production data patterns
5. **Test all states** - ensure fixtures cover edge cases
6. **Clean up** testnet contracts periodically to avoid clutter
7. **Document assumptions** when adding new fixture scenarios

## Troubleshooting

### "Contract deployment failed"
- Ensure your testnet account has sufficient XLM balance
- Check network connectivity to Stellar testnet
- Verify Stellar CLI is properly configured

### "Failed to initialize campaign"
- Check that goal, deadline, and other parameters are valid
- Ensure token contract ID is correct
- Verify creator address has authorization

### "Registry registration failed"
- This is non-fatal - campaign still works without registry
- Check registry contract ID is valid
- Ensure registry contract is deployed and functional

### "Fixture data is stale"
- Regenerate fixtures: `npx ts-node scripts/generate-fixtures.ts`
- Re-seed testnet: `./scripts/seed-testnet.sh`
- Clear cached data and restart dev server
