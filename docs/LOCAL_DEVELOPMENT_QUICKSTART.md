# Local Development Quick Start

Get up and running with Fund-My-Cause development environment in minutes.

## Prerequisites

- **Node.js** 20+ and npm
- **Rust** 1.70+ with `wasm32-unknown-unknown` target
- **Stellar CLI** installed and configured
- **Testnet account** with XLM balance (optional, for contract deployment)

## Quick Setup

### 1. Clone and Install

```bash
git clone https://github.com/Fund-My-Cause/Fund-My-Cause.git
cd Fund-My-Cause
npm install
```

### 2. Build Contracts

```bash
# Build all contracts
npm run contracts:build

# Or manually
cargo build --release --target wasm32-unknown-unknown
```

### 3. Choose Your Development Path

#### Option A: Frontend Development (Mock Data)

Perfect for UI/UX work without deploying contracts.

```bash
# Generate test fixtures
npm run fixtures:generate

# Start frontend
cd apps/interface
npm run dev
```

Open http://localhost:3000

The frontend will use mock data from `fixtures/test-fixtures.json`.

#### Option B: Full Stack Development (Real Contracts)

Best for testing contract integration and full functionality.

**Step 1: Setup Testnet Credentials**

```bash
# Generate testnet account (if needed)
stellar keys generate --network testnet --name testnet-deployer

# Fund account from friendbot
stellar keys address testnet-deployer
# Visit: https://friendbot.stellar.org/?addr=YOUR_ADDRESS

# Set environment variables
export CREATOR=$(stellar keys address testnet-deployer)
export TOKEN="CDLZFC3SYJYDZT7K67VZ75HPJVIEUVNIXF47ZG2FB2RMQQVU2HHGCYSC" # Native XLM or custom token
```

**Windows:**
```powershell
$env:CREATOR = (stellar keys address testnet-deployer)
$env:TOKEN = "CDLZFC3SYJYDZT7K67VZ75HPJVIEUVNIXF47ZG2FB2RMQQVU2HHGCYSC"
```

**Step 2: Seed Testnet**

```bash
# Unix/Linux/Mac
./scripts/seed-testnet.sh

# Windows
.\scripts\seed-testnet.ps1
```

This creates:
- 5 sample campaigns with varied states
- Registry contract
- Configuration in `apps/interface/.env.local`

**Step 3: Start Frontend**

```bash
cd apps/interface
npm run dev
```

Open http://localhost:3000 - connected to real testnet contracts!

## Available NPM Scripts

```bash
# Development
npm run dev                    # Start frontend dev server
npm run build                  # Build frontend for production

# Contracts
npm run contracts:build        # Build Rust contracts to WASM
npm run contracts:test         # Run Rust contract tests

# Testing
npm run test:e2e              # Run Playwright E2E tests
npm run fixtures:generate      # Generate test fixtures JSON

# Seeding (Unix/Linux/Mac only via npm)
npm run seed:testnet          # Seed testnet with sample campaigns

# Code Quality
npm run lint                  # Lint frontend code
npm run format                # Format code with Prettier
```

## Typical Development Workflows

### Frontend Component Development

```bash
# 1. Generate fixtures
npm run fixtures:generate

# 2. Start dev server
cd apps/interface && npm run dev

# 3. Edit components in apps/interface/src/
# 4. Changes hot-reload automatically
```

### Contract Development

```bash
# 1. Edit contract in contracts/crowdfund/src/
# 2. Build
npm run contracts:build

# 3. Test
npm run contracts:test

# 4. Deploy to testnet for integration testing
./scripts/deploy.sh --creator $CREATOR --token $TOKEN \
  --goal 1000000000 --deadline $(date -d '+30 days' +%s) \
  --title "Test Campaign"
```

### E2E Test Development

```bash
# 1. Generate fixtures (or seed testnet)
npm run fixtures:generate

# 2. Start dev server
cd apps/interface && npm run dev

# 3. Run tests (in another terminal)
npm run test:e2e

# 4. Run tests in UI mode for debugging
npm run test:e2e -- --ui

# 5. Edit tests in e2e/
```

### Full Integration Testing

```bash
# 1. Seed testnet with real contracts
./scripts/seed-testnet.sh --creator $CREATOR --token $TOKEN --num-campaigns 10

# 2. Start frontend (uses .env.local from seed script)
cd apps/interface && npm run dev

# 3. Manually test flows in browser
# 4. Run E2E tests against real contracts
npm run test:e2e
```

## Common Issues

### "stellar: command not found"

Install Stellar CLI:
```bash
cargo install --locked stellar-cli --features opt
```

### "Failed to deploy contract"

- Ensure testnet account has XLM balance (use friendbot)
- Check network connectivity
- Verify Stellar CLI is configured: `stellar network ls`

### "Contract initialization failed"

- Verify goal > 0 and deadline is in future
- Check token contract ID is valid
- Ensure creator address matches deployer

### Frontend shows "No campaigns found"

- If using fixtures: run `npm run fixtures:generate`
- If using testnet: run `./scripts/seed-testnet.sh`
- Check `.env.local` has correct contract IDs
- Verify contracts are deployed: `stellar contract info --id CONTRACT_ID --network testnet`

### E2E tests fail

- Ensure dev server is running on http://localhost:3000
- Check fixtures are generated or testnet is seeded
- Run in headed mode to debug: `npm run test:e2e -- --headed`
- Check test output and screenshots in `test-results/`

## Tips for Productive Development

1. **Use fixtures for rapid UI iteration** - No need to deploy contracts for UI work
2. **Seed testnet once, reuse contracts** - Contracts persist until you re-seed
3. **Run E2E tests before PRs** - Catch integration issues early
4. **Keep .env.local.backup** - Seed script creates backups automatically
5. **Use verbose mode** - Add `--verbose` or `-v` to seed scripts for debugging
6. **Generate fresh fixtures** - Run `npm run fixtures:generate` before test sessions
7. **Monitor Horizon** - Check https://horizon-testnet.stellar.org for contract state
8. **Clean up periodically** - Archive old seed data to keep fixtures/ tidy
